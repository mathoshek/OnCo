Introducere

Aceasta lucrare propune o solutie pentru paradigma client server in cadrul tehnologiilor web, prin realizarea unei aplicatii ce gestioneaza contactele personale online. Pe baza unui login sau prin crearea unui nou count valid, clientii se vor conecta la un server web pe baza unui login, acest avand posibiltatea sa gestioneze propria agenda dupa bunul plac. Clientul va fi capabil sa adauge contacte, sa stearga contacte, sa caute contacte dupa anumite criterii (job, varsta, nume, prenume etc) Accentul se pune pe cererile clientului şi modul în care sunt tratate acestea si cum sunt rezolvate de catre server.

Tehnologii

HTML 5 si CSS3

HTML5 este esential in dezvoltarea aplicatiei deoarece este un limbaj pentru structurarea si prezentarea continutului pentru web Cu ajutorul acestui limbaj se creaza interfata grafica, partea vizuala a aplicatiei. Fiecarui tag ii este asociat un id unic, astfel incat pentru fiecare tag in partea sa i se poata aplica anumite modele grafice cu ajutorul CSS-ului. Aceste doua limbaje lucreaza foarte bine impreuna, asftel incat HTML-ul si CSS-ul utilizate impreuna pot oferi un mediu online cat mai atractiv, interesant dar si placut in acelasi timp.

PHP

Pentru partea de backend a aplicatie se recurge la un limbaj precum este PHP. Ne ajuta la creare unei sesiunei pentru un utilizator : clientul se conecteaza, se deschide o sesiune, aceste solicita cereri din partea serverului, iar in momentul cand s-a deconectat aceasta sesiune este inchisa; ofera posibilitatea de creare de clase si de instantiere a obiectelor a acestor clase, pentru a veni in ajutorul programatorului de a scrie cod modularizat astfel orice solicitare de update sa se poata realiza in cel mai scurt timp, ofera anumite functii predefinite care vin in ajutorul dezvoltarii acestei aplicatii.

Mongo Database

Pentru stocarea si managementul datelor, se utilizeaza MongoDB. MongoDB este o baza de date NoSQL open-source orientata pe documente. Diferenta principala consta in faptul ca stocarea datelor nu se face folosind tabele precum intr-o baza de date relationala, MongoDB stocheaza datele sub forma de documente JSON cu schemes dinamice. Acest tip de baza de date vine cu urmatoarele facilitati: + baza de date stocheaza obiecte. Aceste documente reduc forma de join + lipsa joinurilor face ca interogarile distribuite sa fie rapide + indexarea se face asupra cheilor de regasire ale documentelor + MongoDB foloseste map/reduce pentru agregare si procesare
